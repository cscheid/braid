# braid-viewer — Tauri v2 desktop app

Thin Rust shell over `braid-config`: `invoke` commands only — the webview syncs
directly to the sync server (no CRDT traffic through Rust). See `docs/viewer.md`.

## Build & run

- **Use `cargo xtask viewer-dev` / `viewer-build`** (wrap `cargo tauri dev/build`).
  Needs `cargo install tauri-cli --version '^2' --locked`; Linux needs WebKitGTK
  deps (see `docs/viewer.md`).
- **Never `cargo build --release -p braid-viewer` to launch it.** Plain cargo
  omits the `custom-protocol` feature → a dev-mode binary that loads
  `http://localhost:5173` → blank `ERR_CONNECTION_REFUSED`. `cargo build -p
  braid-viewer` is a compile check only (what `viewer.yml` CI runs). Don't add
  `custom-protocol` manually — it's CLI-managed.
- Excluded from `default-members` (musl can't build Tauri), so `cargo xtask ci`
  and plain `cargo build`/`test` skip it. Build/check with `-p braid-viewer`.

## Secret hygiene (non-negotiable)

Doc id is a **bearer secret**. Never log `UiConfig` / `docUrl`. The viewer
registry (`~/.config/braid/viewer.toml`) holds **folder paths only**.
`get_config_cmd` returns the `automerge:` URL, never the raw id.

## Debugging

Startup logs each window's resolved URL to stdout + `braid-viewer.log` (OS log
dir). `localhost:5173` there = you built a dev-mode binary (see Build & run).
Release exe has devtools (right-click → Inspect) and no console on Windows, so
the log file is the trace.

## Gotchas

- Main window is built **programmatically in `setup`** to pin `data_directory`
  (offline cache), so `tauri.conf.json` has `"windows": []`. Don't re-add one.
- `gen/schemas/*.json` are **generated by `tauri-build`** from `capabilities/` —
  don't hand-edit; regenerate by building (commit the result).
- `time` is pinned `=0.3.47` (Rust ≥1.94 coherence vs tauri). Don't unpin.
- Viewer cache (IndexedDB) is **independent** of the CLI's `samod` cache
  (`~/.cache/braid`) — separate replicas converging via the sync server.
- CSP `connect-src` base is `wss://sync.automerge.org` (static, in
  `tauri.conf.json`). Non-default servers (registered projects' `sync_server` +
  `allowed_sync_servers`) are appended at runtime in `setup` via
  `on_web_resource_request` → `csp::augment_connect_src`. The allowlist is a
  **startup snapshot**: a project added at runtime on a non-default server needs
  a restart before it can sync.
